PluginPulse Web Tool

Add an auto-updater to a compiled Paper/Spigot plugin — no source code, no build tools. Everything runs in your browser. Your jar is never uploaded anywhere.

Not what you need?

  • I build my own plugin → shade the library.
  • I run a server and just want my plugins updated → the companion plugin.
  • You're in the right place if you have a jar but can't or won't rebuild it.

Limits

    
    
      
  • Uses the wrapper strategy. If the main class is final (the default for Kotlin
        plugins), the tool clears that flag so the wrapper can attach — this only permits subclassing and is safe for
        a plugin main class. For editing final mains in place instead, the CLI pluginpulse-inject uses the
        instrument strategy.
    • 
      
  • Works on any class-file version (including Java 25 / mc26), legacy plugin.yml and
        paper-plugin.yml.
    • 
      
  • It can't confirm the jar runs — always test before sharing.
